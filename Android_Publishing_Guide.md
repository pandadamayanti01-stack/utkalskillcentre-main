# 📱 Google Play Store Publishing Guide: Trusted Web Activity (TWA)

This guide provides step-by-step instructions for packaging **Utkal Skill Centre** as a high-performance Android App Bundle (`.aab`) using the official **Trusted Web Activity (TWA)** standard and publishing it to the Google Play Store.

---

## 🏗️ How TWA Works
A **Trusted Web Activity** opens your live website (hosted securely on Firebase) in a full-screen, native browser frame. 
* **Benefits**: No browser URL bar, automatic offline handling, and **instant updates** (whenever you deploy code updates via `npm run deploy`, they appear in the app instantly without store resubmission!).
* **Verification**: Google matches your website’s security credentials against your app's digital signature. Once they match, the address bar disappears completely.

## 🗺️ Step 1: Prepare Your Project
We have already pre-configured the following:
1. **PWA Manifest**: Configured inside `vite.config.ts` and `public/manifest.json`.
2. **Offline Support**: Configured using `vite-plugin-pwa` (Workbox cache strategy).
3. **Asset Links Placeholder**: Created at [assetlinks.json](file:///d:/WebApp/utkalskillcentre-main/public/.well-known/assetlinks.json).

## 🛠️ Step 2: Package with PWABuilder (Easiest Way)

[PWABuilder](https://www.pwabuilder.com/) is a free, open-source tool built by Microsoft that compiles PWAs into high-quality store bundles.

1. **Deploy your latest code** to production:
   ```bash
   npm run deploy
   ```
2. Go to **[PWABuilder.com](https://www.pwabuilder.com/)**.
3. Enter your live, deployed website URL (e.g., `https://utkalskillcentre.web.app` or your custom domain) and click **Start/Test**.
4. Once the PWA report card completes (it should score 100/100!), click **Package for Store**.
5. Select **Android** and click **Generate Package**.
6. A form will appear. Fill in the following fields:
   * **Package ID**: Use a unique identifier (e.g., `com.utkalskillcentre.app` or `com.utkalskillcentre.twa`).
   * **App Name**: `Utkal Skill Centre`
   * **Short Name**: `Utkal Skill Centre`
   * **Start URL**: `/`
7. Click **Download Package**.

### 📦 What's inside the downloaded ZIP?
* `app-release.aab`: The final production bundle file you will upload to the Google Play Console.
* `signing-key.keystore`: The key file used to digitally sign your app. **CRITICAL: Store this in a secure backup location!** If you lose this key, you will never be able to update your app in the Google Play Store.
* `assetlinks.json`: A file containing the SHA-256 fingerprint of the key generated for you.

## 🔒 Step 3: Complete the Digital Handshake

To prove you own the domain and remove the browser URL bar:

1. Open your downloaded `assetlinks.json` file. It will contain a structure similar to this:
   ```json
   [
     {
       "relation": ["delegate_permission/common.handle_all_urls"],
       "target": {
         "namespace": "android_app",
         "package_name": "YOUR_PACKAGE_ID",
         "sha256_cert_fingerprints": [
           "XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX:XX"
         ]
       }
     }
   ]
   ```
2. Copy this content.
3. Open [public/.well-known/assetlinks.json](file:///d:/WebApp/utkalskillcentre-main/public/.well-known/assetlinks.json) and replace the placeholder text with your actual **Package ID** and **SHA-256 Cert Fingerprint** generated by PWABuilder.
4. Deploy the file live to your Firebase Hosting:
   ```bash
   npm run deploy
   ```
5. Verify it is public by opening this URL in your browser:
   `https://<your-domain>/.well-known/assetlinks.json`

> [!NOTE]
> If you choose to let Google Play manage your app signing keys (strongly recommended!), Google will generate a *new* SHA-256 fingerprint for production. 
> You can find this inside the **Google Play Console** under **Setup** -> **App Integrity**. Copy the **SHA-256 certificate fingerprint** from there and add it to the `sha256_cert_fingerprints` array in your `assetlinks.json` so both your local test key and Google's production key are trusted!

## 🚀 Step 4: Google Play Store Submission

1. **Create Developer Account**: Go to the [Google Play Console](https://play.google.com/console) and complete registration ($25 USD).
2. **Create a New App**:
   * App Name: `Utkal Skill Centre`
   * Default Language: `English` (or your preferred default)
   * App or Game: `App`
   * Free or Paid: `Free`
3. **Complete the Initial Setup Tasks**:
   * **App Access**: Choose "All functionality is available without special access" (or provide test credentials if you have locked student-only areas).
   * **Content Rating**: Complete the questionnaire (choose "Educational" category).
   * **Privacy Policy**: Provide a link to your privacy policy (you can host this on your Firebase site).
   * **Target Audience**: Select age group (e.g., 13-15, 16-17, 18 and over) and answer standard questions.
4. **Create a Production Release**:
   * Go to **Release** -> **Production** in the sidebar.
   * Click **Create New Release**.
   * Drag and drop the `app-release.aab` bundle file here.
   * Fill out the Release Notes (e.g., `"Initial release of Utkal Skill Centre mobile application."`).
5. **Upload Store Assets**:
   * Upload an App Icon (512x512 PNG).
   * Upload a Feature Graphic (1024x500 PNG).
   * Upload Phone & Tablet Screenshots (at least 2 to 4 high-quality screenshots showing the login page, dashboard, and MCQ features).
6. **Submit for Review**: 
   * Click **Review Release** and then click **Start Rollout to Production**.
   * It takes **2 to 4 business days** for Google's trust and safety team to approve your initial release.

## 💡 Troubleshooting & Best Practices

* **Address Bar is still showing**:
  * Double check that `https://<your-domain>/.well-known/assetlinks.json` is perfectly valid JSON and doesn't return a 404.
  * Check that the SHA-256 signature in `assetlinks.json` matches the signing certificate of the package installed on the phone.
  * Wait a few minutes; sometimes the Android system caches old asset verification records.
* **Firebase redirection issues**:
  * In [firebase.json](file:///d:/WebApp/utkalskillcentre-main/firebase.json), we have rewrite rules pointing everything to `/index.html`. This is correct for Single Page Apps (SPAs), and it will automatically serve `.well-known/assetlinks.json` since it exists as a physical file in the public directory (static files override rewrite rules).
